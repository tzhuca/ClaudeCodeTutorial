# 第一章：Commands 思维模型（核心🔥）

## 💡 三层控制系统

| 类型 | 符号 | 本质作用 |
|------|------|----------|
| Slash 命令 | / | 控制 AI |
| Shell 命令 | ! | 控制系统 |
| 文件引用 | @ | 提供上下文 |

👉 记住一句话：

> **/ 决定怎么做，@ 决定看什么，! 决定执行什么**

---

## 🧪 练习

理解下面指令：

```
@server.js 修复 bug
! npm run dev
```

---

# 第二章：Slash 命令（控制 AI）

## 📌 基础控制

| 命令 | 作用 | 使用时机 |
|------|------|----------|
| /help | 查看帮助 | 不熟悉命令 |
| /exit | 退出 | 结束任务 |
| /clear | 清空上下文 | 开新任务前 |
| /compact | 压缩上下文 | Token过高 |

---

## 📌 上下文管理

| 命令 | 作用 |
|------|------|
| /context | 查看上下文 |
| /cost | 查看消耗 |

👉 推荐习惯：

```
/context
```

---

## 📌 会话管理

| 命令 | 作用 |
|------|------|
| /resume | 恢复会话 |
| /export | 导出记录 |
| /rename | 重命名 |

---

## 🧪 实战练习

执行：

```
/help
/context
/compact
```

思考：
- Token 是否变化？

# 第三章：Shell 命令（控制系统）

## 📌 基本用法

```bash
! npm install
! python main.py
```

Claude 会：
1. 执行命令
2. 读取输出
3. 自动继续处理

---

## ⚠️ 使用原则

- 只执行你理解的命令
- 避免删除/覆盖文件

---

## 🧪 练习


# 第四章：文件引用（提供上下文）

# 第五章：命令组合（最重要🔥）

## 🎯 为什么重要？

单个命令 ≠ 实际开发

👉 真正能力是：组合使用

---

## 📌 标准组合流程

```
@代码文件
描述需求
!运行验证
```

---

## 📌 示例1：修复 bug

```
@index.js 修复报错
! npm run dev
```

---

## 📌 示例2：功能开发

```
@api.js 增加登录接口
! node server.js
```

---

## 🧪 练习

自己尝试：

```
@文件
修改功能
!运行
```

# 第七章：常见错误与解决

## ❌ Slash 命令无效

原因：未进入交互模式

解决：

```bash
claude
```

---

## ❌ 上下文过大

解决：

```
/compact
```

---

## ❌ 命令执行失败

原因：环境问题

解决：

- 检查 Node/Python
- 手动执行一次命令
